# Homepage accessibility audit — 2026-05

이 리포트는 `aitc.dev` landing page (`apps-in-toss-community.github.io`)의
자동화 가능한 accessibility 검수 결과를 기록한다. 사람이 OS 음성 합성을
켜야 가능한 스크린리더(VoiceOver/NVDA) 검수는 자동화 범위 밖이라
체크리스트로 인계한다.

> **Re-verified 2026-05-11** — harness(`e2e/a11y.spec.ts`)를 로컬에서
> 다시 돌려 12/12 통과(axe 6 × 라이트/다크 + 구조 3 + skip-link 3),
> Playwright MCP로 `/`, `/ko/`, `/en/` 콘솔/접근성 트리를 교차 확인.
> axe 위반은 여전히 0, 아래 "부수적으로 발견" 섹션의 React #418
> hydration mismatch도 그대로 재현되어 (`assets/client-*.js`) 별도
> 이슈로 추적 중. 코드 변경 없음.

## 페이지 구조 요약 (`/ko/` 기준)

라우트가 다 동일한 컴포넌트 트리이므로 한 라우트만 캡처. 외부 링크는
모두 `<span class="sr-only"> (새 탭에서 열기)</span>` cue를 가지고 있다.

```
banner (header)
  link "apps-in-toss-community" → /ko/
  button "테마: 시스템 (클릭하여 라이트로 전환)"   ← ThemeToggle (aria-label 동적)
  navigation "Language switcher"
    link "한국어" (aria-current="page")
    link "English"

main
  section (Hero)
    heading "apps-in-toss-community" [level=1]
    link "웹 데모 열기 (새 탭에서 열기)"
    link "GitHub → (새 탭에서 열기)"

  region "이런 것들을 할 수 있어요" (ValueList)
    heading [level=2] + list

  region "사용 가능" (ProjectGrid available)
    heading [level=2] + 4× article
      heading [level=3] "@ait-co/devtools (새 탭에서 열기)"
      …

  region "예정" (ProjectGrid coming-soon)
    heading [level=2] + 3× article (Coming Soon chip)

  region "devtools로 시작하기" (QuickStart)
    heading [level=2] + paragraphs + code blocks
    button "Copy to clipboard"

  region "리소스" (Resources)
    heading [level=2] + list of external links

contentinfo (footer)
  link "community-driven (opens in new tab)"
```

H1 한 개, h2/h3 hierarchy 정상, landmark 4종(`banner`, `main`, `contentinfo`,
+ navigation/region들) 모두 존재. 모든 region은 `aria-labelledby`로 heading에
연결되어 있어 스크린리더 rotor에서 의미 있는 이름이 노출된다.

## 발견된 axe 위반

**없음.** 이전 a11y pass 1–3 (PR #30, #31, #33)에서 도입한 정책이 효과적이었다:

- skip-to-main 링크 (`Layout.tsx`)
- 모든 외부/`target="_blank"` 링크의 `.sr-only` "(새 탭에서 열기)" cue
- ProjectCard 별 모양·STAR count의 `aria-hidden`/`sr-only` 분리
- Resources/ValueList/ProjectGrid/QuickStart의 `aria-labelledby`
- ThemeToggle/LangBanner dismiss button의 동적 `aria-label`
- LangBanner를 `<section aria-label>`로 둬서 `<header>`의 implicit `role="banner"`와 충돌 회피
- WCAG AA color-contrast — `--cta-bg` / `--link-fg` / `--code-label-fg` /
  업데이트된 `--muted` 토큰 (light & dark) — 브랜드 `--accent`는 그대로 유지

따라서 이 PR은 **코드 변경 없이** audit 인프라(테스트, CI, 문서)만 추가한다.

## 후속 — 사람이 직접 해야 하는 검수

자동화로 못 잡는 부분. 시간 날 때 1회씩 돌려보고 본 문서에 결과 추가.

### keyboard-only navigation (15분)

- [ ] Tab 1번에 skip-to-main 링크 포커스 (ko/en/`/` 모두)
- [ ] skip-to-main에서 Enter 누르면 `<main>` 내용으로 점프
- [ ] Header 진입 → brand → ThemeToggle → 한국어 → English 순서
- [ ] ThemeToggle Space/Enter 누르면 system→light→dark→system 순환
- [ ] Hero CTA 두 개 모두 Enter로 외부 링크 열림
- [ ] Project card 링크 Tab 순서 자연스러움
- [ ] QuickStart Copy 버튼 Enter로 동작 + visual feedback
- [ ] LangBanner의 dismiss(×) 버튼 Enter로 닫힘
- [ ] Tab을 끝까지 누르면 footer "community-driven" 링크에서 마침
- [ ] focus indicator(outline)가 모든 interactive 요소에서 보임
- [ ] light/dark 모두 focus indicator 시인성 충분

### VoiceOver (macOS, 20분)

- [ ] Cmd+F5로 켜기 → 페이지 로드 시 `<title>` 읽음
- [ ] VO+U → Landmarks rotor에 banner / main / contentinfo / navigation / region이 의미 있는 이름으로 나열됨
- [ ] VO+U → Headings rotor에 h1/h2/h3 hierarchy가 한 번에 보임
- [ ] 외부 링크에서 "새 탭에서 열기"가 실제로 읽힘 (cue가 visual에서는 가려져 있어도 음성에는 들려야 함)
- [ ] ThemeToggle을 누를 때 `aria-label`이 mode에 따라 갱신 ("테마: 라이트…" 등)
- [ ] LangBanner가 보이는 상황에서 dismiss button "Dismiss language banner"로 읽힘
- [ ] ProjectCard "Coming Soon" chip이 heading 안에 있어도 헷갈리지 않음
  - heading text는 "{repo} (새 탭에서 열기) Coming Soon"으로 합쳐 읽힘 — 의도된 동작
- [ ] `/en/` 페이지에서 영어 음성으로 읽힘 (`<html lang="en">` 적용 확인)
- [ ] `/ko/` 페이지에서 한국어 음성으로 읽힘 (`<html lang="ko">`)

### NVDA (Windows, 20분)

- [ ] Insert+F7 (Elements list) → Headings 탭에 의미 있는 트리
- [ ] Insert+F7 → Landmarks 탭에 의미 있는 region 이름
- [ ] focus mode/browse mode 전환 시 ThemeToggle/Copy 버튼이 정상 동작
- [ ] LangBanner dismiss 버튼이 "Dismiss language banner button"으로 읽힘
- [ ] Korean/English 페이지에서 음성 합성기가 적절히 전환

### `prefers-reduced-motion`

axe로 직접 검증 안 됨. macOS System Settings → Accessibility → Display →
Reduce Motion ON 후 다음 확인:

- [ ] LangBanner 표시/숨김 transition이 즉시 일어나거나 비활성
- [ ] ThemeToggle hover/active transition이 즉시 일어나거나 비활성
- [ ] CSS `@media (prefers-reduced-motion: reduce)` 룰이 모든 transition을
      덮고 있는지 인덱스 CSS 검토
